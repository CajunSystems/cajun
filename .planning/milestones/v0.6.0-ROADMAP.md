## Phase Details

### Phase 12: Upgrade & Compatibility ✅
**Goal:** Bump `roux` to `0.2.1`, verify full build, fix breaking changes.

Plans: 12.1 (3 tasks)

**Key findings:**
- Roux v0.2.0 changed `compose()` to catch `UnsupportedOperationException` (not `ClassCastException`). Handlers with narrow type params throw CCE → now propagates uncaught.
- Fix: all handlers that participate in `compose()`/`orElse()` must implement `CapabilityHandler<Capability<?>>` and throw UOE for unhandled types.
- Preferred fix: `CapabilityHandler.builder().on(Type.class, fn).build()` — auto-throws UOE for unregistered types.
- `DefaultEffectRuntime` is now `AutoCloseable` — inheriting `close()` would shut down ActorSystem's executor. Override as no-op.

---

### Phase 13: Bridge — Concurrency & Timeout ✅
**Goal:** Test `Effects.parAll/race/traverse` and `timeout(Duration)` through `ActorEffectRuntime`. Verify v0.2.1 scoped-fork fix.

Plans: 13.1 (4 tasks)

**Key findings:**
- Roux `TimeoutException` is `com.cajunsystems.roux.exception.TimeoutException`, not `java.util.concurrent.TimeoutException`. Use `getClass().getName().contains("TimeoutException")` for coupling-free assertion.
- Java 21 does not allow `sealed interface` inside a method body — define as static nested type in test class.
- Roux v0.2.1 fix confirmed: `forkIn(scope)` now inherits parent `ExecutionContext` including capability handlers.

---

### Phase 14: Modernize Retry & Error Examples ✅
**Goal:** Replace hand-rolled retry with built-in `RetryPolicy`. Add `tap()`/`tapError()`. New timeout example.

Plans: 14.1 (4 tasks)

**Key findings:**
- `retry(n)` counting: n = ADDITIONAL attempts (not total). `retry(2)` = 3 total.
- `retryWithDelay()` and `retry(RetryPolicy)` widen error type to `Throwable`.
- `tap()` fires on success only; passes value through unchanged.
- `tapError()` fires on failure only; re-throws original error (observe-and-rethrow, NOT recovery).
- `timeout().catchAll(...)` produces `Effect<Throwable, A>` — test methods need `throws Throwable`.

---

### Phase 15: New Concurrency & Resource Examples ✅
**Goal:** Demonstrate `Effects.parAll/race/traverse` and `Resource<A>` in effect actor patterns.

Plans: 15.1 (3 tasks)

**Key findings:**
- `resource.use()` and `Resource.ensuring()` widen to `Throwable` — test methods need `throws Throwable`.
- `traverse()` does NOT widen — stays as `E`.
- `Resource.fromCloseable(effect)` is the clean path for `AutoCloseable`; no release lambda needed.
- `Resource.make(acquire, release)` gives full control; release always runs on success AND failure.
- `Resource.ensuring(effect, finalizer)` is try-finally pattern; finalizer runs regardless of outcome.

## API Knowledge Base (v0.2.1)

| Pattern | Detail |
|---------|--------|
| Handler compose contract | `CapabilityHandler<Capability<?>>` required; throw UOE for unhandled types; use `.builder()` |
| ActorEffectRuntime.close() | No-op — executor owned by ActorSystem, not runtime |
| Roux TimeoutException | `com.cajunsystems.roux.exception.TimeoutException` — check class name, not instanceof |
| retry(n) counting | n = additional attempts; `retry(2)` = 3 total (1 initial + 2 retries) |
| retryWithDelay / RetryPolicy | Widen error to Throwable; test methods need `throws Throwable` |
| tap() | Success-side only; passes value through; does not alter error type |
| tapError() | Failure-side only; observe-and-rethrow; does NOT recover |
| timeout().catchAll() | Produces `Effect<Throwable, A>`; test needs `throws Throwable` |
| traverse() | Sequential; error stays as E (NOT widened) |
| parAll / race | Error widens to Throwable |
| Resource.use() | Widens to Throwable |
| Resource.ensuring() | Widens to Throwable; finalizer always runs |
| Resource.fromCloseable() | Shorthand for AutoCloseable; release = close() |
| Scoped fork | Roux v0.2.1 fix: forkIn(scope) inherits parent ExecutionContext + capability handlers |
| Sealed interfaces in tests | Java 21: sealed interfaces must be static nested types, not local |
